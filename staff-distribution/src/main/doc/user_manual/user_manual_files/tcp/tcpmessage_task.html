tcpMessage(deprecated)

Description

TcpMessage type represents an ITCH message object.

Parameters

Attribute Description Default Required
type Type of the message specified in the dictionary. Note: this parameter is case sensitive. None Yes
id Reference name of the message objest to use with other tasks. None No
source Reference name of the message objest specified earlier used in other tasks. None No

Nested Elements

tcpMessage supports nested elements <field> and <group>.

field

<Field> element represents ITCH message field.

Attribute Description Default Required
name Name of the ITCH field specified in the dictionary. Set of supported names depends on the parser library used for testing. None Yes
value Field value. Default field value is specified in the dictionary and depends on the field type. None No
tolerance Specifies accuracy of comparison of expected value with received one. If difference of values exceeds specified tolerance, then comparison fails. None No

group

<Group> element represents ITCH message repeating group.

Attribute Description Default Required
name Name of the ITCH repeating group specified in the dictionary. Set of supported names depends on the parser library used for testing. None Yes

entriesNum

<entriesNum> element represents number of entries in the specified repeating group. The element can be used with <tcpReceive> task only.

Attribute Description Default Required
name Name of the ITCH repeating group specified in the dictionary. Set of supported names depends on the parser library used for testing. None Yes
values Expected number of entries in the repeating group. None Yes

<Group> element supports <groupentry> nested element, which supports <field> and <group> nested elements. Leading field of the group is set automatically according to the number of specified group entries.

Examples

Message with group

	<tcpMessage type="U" id="UserAccountResponse">                                  
		<field name="UserSessionID" value="5"/>
		<group name="UserAccounts">                                             
			<groupentry>                                                    
				<field name="AccountID"		value="1"/>
				<field name="AccountShortName"	value="Acc1"/>          
				<field name="AccountName"	value="Test account 1"/>
				<field name="ClearingFirmID"	value="1"/>
				<field name="ClearingFirmName"	value="ClrngFirm1"/>
				<field name="OrganizationID"	value="1"/>
				<field name="OrganizationName"	value="Org1"/>
			</groupentry>
			<groupentry>
				<field name="AccountID"		value="2"/>
				<field name="AccountShortName"	value="Acc2"/>
				<field name="AccountName"	value="Test account 2"/>
				<field name="ClearingFirmID"	value="1"/>
				<field name="ClearingFirmName"	value="ClrngFirm1"/>
				<field name="OrganizationID"	value="1"/>
				<field name="OrganizationName"	value="Org1"/>
			</groupentry>
		</group>                                                    
	</tcpMessage>

Using source and tolerance

	<tcpMessage type="D" id="NewOrderSingle">
		<field name="ClOrdID" value="ClOrd${ID}" />
		<field name="InstrumentID" value="${Symbol}" />
		<field name="TransactTime" value="${ID}" tolerance="1000"/>
		<field name="OrderClass" value="R" />
		<field name="OrderType" value="2" />
		<field name="Side" value="${Side}" />
		<field name="OrderQty" value="${Qty}" />
		<field name="HiddenQty" value="0" />
		<field name="Price" value="${Price}" />
		<field name="TimeInForce" value="0" />
		<field name="ExpireTime" value="0" />
		<field name="AccountID" value="${AccountID}" />
		<field name="UserID" value="123" />
	</tcpMessage>
	<tcpReceive refid="TClient" timeout="10">
		<tcpMessage source="NewOrderSingle" />
	</tcpReceive>
